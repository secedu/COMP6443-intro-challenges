This is an example OAuth provider. The provider is the thing you sign in with. For example, if you pressed "Sign in with Google", then "Google" would be your OAuth provider.

The responsibility of the OAuth provider is to send back some information (in this case a code) to the OAuth app. The OAuth app will use this code to associate the user's session with their OAuth provider account.

For example, if Google was the OAuth provider, and example.com was the client application, example.com might have a "Sign in with Google" login option. If the user presses it, they will be redirected to Google.
Google will allow them to sign in and afterwards will give a code to example.com. It does this by redirecting your browser to a URL on example.com, for example https://example.com/callback?code=AUTH_CODE_HERE.
The browser landing on this URL will cause a HTTP GET request to the example.com backend. The example.com backend now has the code in the URL, which can be sent to Google servers in exchange for information about the user, such as their name or email address.

For this example OAuth application, we are using {{ auth_provider_url }} as the OAuth provider and {{ client_app_url }} as the client app. Intercept the requests to see if you can understand how OAuth works!
